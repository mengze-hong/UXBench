📚 About UXBench

Why real user feedback changes everything

🎯 Real-User Signal

Unlike benchmarks relying on expert annotation or synthetic prompts, UXBench is mined from 38,331 disliked and 33,253 liked real conversations — capturing what actual users find frustrating or satisfying.

⚖️ Balanced Evaluation

Our core metric — Avg% = (Good-Acc + Bad-Acc) / 2 — rewards models that can identify both good and bad responses, exposing the systematic positive bias present in all current LLM judges.

🔮 Multi-Task Coverage

Three complementary tasks: UX Judge (discriminative), UX Eval (comparative scoring), and UX Recovery (generative fix) — covering the full quality-assurance cycle.

🥳 Failure & Success Taxonomy

A human-validated taxonomy of 10 failure dimensions and 8 success patterns covering 8 interaction scenarios and 83 domains, enabling fine-grained capability diagnosis.

🧴 Auto-Labeling Pipeline

A 6-stage LLM-powered pipeline processes raw conversations through signal extraction, pre-filtering, miner agent, judge agent, QA scan, and golden set construction — ~120K LLM calls in total.

📊 Training Data

Beyond evaluation, UXBench provides 38,300 training examples with structured failure/success annotations, enabling fine-tuning of reward models and LLM judges for production UX quality assessment.

🔍 Key Findings

Insights from evaluating 26 frontier LLMs on real-user UX data

📊 Systematic Positive Bias

Every model shows Good-Acc ≫ Bad-Acc, with gaps of 27–88 pp. LLMs are biased toward validating responses — a critical flaw for QA pipelines.

🏠 Self-Preference Effect

LLM judges exhibit measurable in-group favoritism: same-family models score each other higher. Mixed-vendor ensembles produce more calibrated evaluations.

🔀 Capability ≠ UX

General-capability rankings show weak correlation (r=0.31) with UXBench rankings. Strong reasoning does not guarantee a good user experience.

🏆 Best Balanced Judge

Claude Opus 4.7 achieves 75.3% Avg-Acc — the only frontier model with Bad-Acc above 60%, showing better resistance to positive bias.

★ Trained Model Wins

Our Pointwise GRM (77.2% Avg-Acc) outperforms all frontier LLMs on Task 1, with Good-Acc 82.1% and Bad-Acc 72.4% — far more balanced.

🔧 Recovery is Hard

Task 3 recovery rates top out at 12.8% (Claude Opus 4.6), revealing that generating genuinely improved responses remains an open challenge even for frontier models.
