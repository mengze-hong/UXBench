# GRM Judge Scripts

This directory contains the Pointwise GRM judge scripts used to score model responses
for Task 2 (UX Eval) and Task 3 (UX Recovery).

## Overview

The **Pointwise GRM** (`pointwise_grm_ux_v031`) is a generative reward model trained
on real user feedback signals. It scores AI responses by predicting whether a user
would find them satisfying ("好") or unsatisfying ("差"), and computes:

```
Score = P(好) / (P(好) + P(差))    # derived from output logprobs
Verdict: 1 = 好 (good), -1 = 差 (bad)
```

The GRM must be served locally via **vLLM** before running these scripts.

---

## Setup: Serving the GRM

The GRM is a HunyuanV3 (20B active params) model requiring 8× H20 GPUs.

```bash
# Set model path (obtain from paper authors or HuggingFace if released)
export MODEL_PATH=/path/to/pointwise_grm_ux_v031/huggingface
export GRM_AUTH_TOKEN=your_token_here   # set any token you choose

# Serve with vLLM (>=0.13.0 required for HYV3ForCausalLM)
export VLLM_WORKER_MULTIPROC_METHOD=spawn
export TOKENIZERS_PARALLELISM=false

nohup vllm serve "$MODEL_PATH" \
    --served-model-name pointwise_grm_ux_v031 \
    --host 0.0.0.0 \
    --port 8021 \
    --tensor-parallel-size 8 \
    --max-model-len 16384 \
    --max-num-seqs 32 \
    --gpu-memory-utilization 0.90 \
    --enforce-eager \
    --trust-remote-code \
    --enable-prefix-caching \
    >> vllm.log 2>&1 &

# Health check (wait ~5 min for startup)
curl http://localhost:8021/v1/models
```

## Task 2: UX Eval — GRM Judge

**Input:** `experiments/results/task2/responses/<model_key>.jsonl`  
Each file contains the responses generated by the model under test:
```json
{"cid": "...", "model": "claude_opus_47", "generated_response": "..."}
```

**Output:** `experiments/results/task2/grm_judge/judge_<model>.jsonl`
```json
{"cid": "...", "judged_model": "...", "verdict": 1, "score": 0.72, "good_logprob": -0.12, "bad_logprob": -2.45}
```

**Run:**
```bash
# Set auth token
export GRM_AUTH_TOKEN=your_token_here

# Judge all models
python scripts/grm_judge/run_grm_judge_task2.py --workers 80

# Judge specific models
python scripts/grm_judge/run_grm_judge_task2.py --models claude_opus_47 gpt_5

# Smoke test (10 samples per model)
python scripts/grm_judge/run_grm_judge_task2.py --smoke 10

# Custom endpoint (multi-node)
python scripts/grm_judge/run_grm_judge_task2.py --endpoint http://<NODE_IP>:8021/v1/chat/completions
```

## Task 3: UX Recovery — GRM Judge

**Input:** `experiments/results/task3/responses/<model_key>.jsonl`  
Each file contains recovery responses. Records must include `user_complaint` and
`generated_response` fields (plus optional `history`, `failed_response`).

**Output:** `experiments/results/task3/grm_judge/judge_<model>.jsonl`

**Run:**
```bash
export GRM_AUTH_TOKEN=your_token_here

python scripts/grm_judge/run_grm_judge_task3.py --workers 60

python scripts/grm_judge/run_grm_judge_task3.py --models claude_opus_47 --smoke 5
```

## Workflow: Full Evaluation Pipeline

```
Step 1 — Generate responses:
    python scripts/run_eval.py --task task2_ux_eval --model <model> --workers 10
    # outputs to: experiments/results/task2/<model>/results.jsonl

Step 2 — Copy to responses/:
    cp experiments/results/task2/<model>/results.jsonl \
       experiments/results/task2/responses/<model_key>.jsonl

Step 3 — GRM judge:
    python scripts/grm_judge/run_grm_judge_task2.py --models <model_key>

Step 4 — View metrics:
    cat experiments/results/task2/grm_judge/main_metrics.json
```

---

## Notes

- Both scripts support **resume**: re-running skips already-completed cids
- Truncation retry is automatic on HTTP 400 (input too long): reduces history turns and response length
- `--smoke N` limits each model to N samples (for quick sanity checks)
- Results include both binary verdict and continuous score for downstream analysis
